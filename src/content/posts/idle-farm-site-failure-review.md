## 一、起因

手头有一个小游戏站 `game.455545.xyz`（15 款经典小游戏合集），上线后用 Google Keyword Planner 跑了一次关键词，发现 `idle miner`（闲置矿工）月搜索量 5,000、竞争度低。顺着这个线索往下挖，又发现了 `idle farming game`（500/mo）、`idle farm tycoon`（50/mo）等一系列低竞争长尾词。

思路很直接：**既然 idle miner 有人做而且流量不错（idle-miner.com ~345K/mo），那 idle farm 是不是也有机会？**

于是就开启了这次的 Idle Farm Game 独立站探索之旅。

## 二、关键词数据

使用 Google Keyword Planner（美国地区），总共跑了两轮：

### 第一轮（中国区定位，仅供参考）

| 关键词 | 月搜索量 | 竞争度 |
|:-------|:--------:|:------:|
| idle miner | 5,000 | 低 |
| idle farm | 500 | 低 |

### 第二轮（美国区定位，决策依据）

| 关键词 | 月搜索量 | 竞争度 |
|:-------|:--------:|:------:|
| idle farming game | 500 | 低 |
| idle farm | 500 | 低 |
| idle farm tycoon | 50 | 低 |
| farm idle game | 50 | 低 |
| farm game online | 5,000 | 中 |
| free farming games | 5,000 | 中 |

如果把所有 idle farm 相关长尾词加起来，总搜索量大约 8,000/mo。即使排名第一，乐观估计也只能拿到 ~2,400/mo 的流量（按 30% 点击率算）。

**第一个警示信号出现了：市场太小。**

对比来看，`cookie clicker` 单月搜索量 150,000+，是 idle farm 相关词的 20 倍。这就是为什么 cookieclicker.com 能做到 2.86M/mo 有机流量，而 idle farm 站的上限要低得多。

## 三、开源游戏调研（GitHub 篇）

既然要建站，首先得有个游戏。考虑到独立开发者资源有限，最务实的方案是**找开源游戏定制部署**。

在 GitHub 上找到了 5 个候选项目：

### 1. AliHaine/GameFarmerJS ⭐ 推荐

- **技术栈**：纯 HTML/CSS/JS，无构建工具
- **游戏类型**：像素风农场模拟器（翻地、播种、浇水、收割）
- **协议**：MIT
- **评分**：⭐⭐

**实际体验**：这是一个技术 demo 而非游戏。界面是像素风网格地图，你需要手动点击翻地、播种子、等它长大、再收割。没有升级系统、没有自动产出、没有 idle 要素。与其说是在玩游戏，不如说是在测试引擎。不好玩。

### 2. aaronvanston/farmclicker

- **技术栈**：React + Redux，需要 npm build
- **游戏类型**：Idle clicker（点击赚钱、买种子、卖资源）
- **协议**：未明确标注
- **评分**：⭐⭐⭐

**实际体验**：比 GameFarmerJS 更像一个游戏，有 clicker 的核心循环。但是 React 架构，部署需要构建流程，而且 UI 风格比较过时。没有自动保存。可改但工作量不小。

### 3. M-Karty/Web-Farm

- **技术栈**：3 个文件（HTML/CSS/JS）
- **游戏类型**：点击小麦赚钱
- **协议**：未标注
- **评分**：⭐

**实际体验**：简陋到令人震惊——一个按钮，点一下加 1 块钱，没了。纯概念验证。

### 4. hatimhtm/night-market-tycoon

- **技术栈**：Vite + JS
- **游戏类型**：夜市经营 idle 游戏
- **协议**：非商业授权 ❌
- **评分**：⭐⭐⭐⭐

**实际体验**：这个是几个里面完成度最高的，有完整的 idle progression 系统。但一非农场主题，二非商业授权，不能用。

### 5. koutoftimer/idle-miner

- **技术栈**：纯 JS
- **游戏类型**：挖矿 idle clicker
- **协议**：MIT
- **评分**：⭐⭐⭐⭐

**实际体验**：这个项目的成功是有原因的。idle-miner.com 用它获得了 345K/mo 流量。游戏本身完成度高，有装备系统、自动挖矿、多矿层、升级树。但它不是农场游戏。

## 四、搭建过程

初步选定 GameFarmerJS 作为基础，理由是：
- 纯前端，零构建，直接部署
- 像素画风看起来像那么回事
- MIT 协议，随便改

计划是：**拿 GameFarmerJS 做底子，加上 idle 自动收益机制，让它从一个手动的农场模拟变成 idle farm game。**

然后就踩进了坑里。

### 坑 1：原版游戏本身不好玩

GameFarmerJS 的地图是 75x95 的网格，每个格子要手动操作。翻地、播种、等待生长、回来收割——这一套操作在 Stardew Valley 里是沉浸式体验，在浏览器里就是无聊的点鼠标。

更致命的是**没有 idle 要素**。Idle 游戏的核心是「离开后还在产出」，GameFarmerJS 的设计初衷完全相反：它要求你在屏幕前一直操作。

要把它改成 idle 游戏，等于重写整个经济系统。

### 坑 2：自己写了一个，方向偏了

意识到 GameFarmerJS 不好改之后，我犯了一个典型错误：**没有停下来确认方向，直接开始写代码。**

花了半小时写了一个完整的 idle farm clicker——有 6 种作物、6 种动物、6 项升级、里程碑成就、离线收益、自动存档。技术上看做完了，但：

- 用户想要的是「GitHub 上那个开源游戏」，不是我凭空造的
- 自己写的游戏没有经过市场验证，好不好玩未知
- 这偏离了「拿开源游戏部署」的初衷

### 坑 3：浏览器自动化建仓库

到要部署的时候，又花了大量时间跟 GitHub 的 React 表单搏斗——浏览器自动化创建仓库，一句 "You can't perform that action at this time" 卡了十几分钟。

回头看，这件事本身就说明这个站做得不顺利：**从游戏选择到部署流程，每一步都在走弯路。**

## 五、失败原因分析

整理一下这个方向失败的核心原因：

### 1. 搜索量太小，天花板太低

所有 idle farm 相关词加起来 ~8,000/mo，即使排名第一，乐观 ~2,400/mo 流量。对比 idle-miner（5,000/mo 搜索量，345K/mo 流量），farm 赛道的需求只有 miner 的 1/6。

为什么相差这么大？想想就知道：

- **挖矿（miner）** 是 Idle 游戏最经典的题材。从 Cookie Clicker 开始，玩家对「数字增长」的瘾头在挖矿题材上被反复验证。
- **种地（farm）** 的受众完全不同。种地游戏的玩家想的是经营和装饰，不是数字最大化。Stardew Valley 卖得好，但 idle farm 这个细分确实没人搜。

### 2. 开源游戏质量堪忧

找了一圈，没有一个能直接用的农场 idle 游戏：
- 好玩的不是开源（night-market-tycoon 非商业授权）
- 开源的不好玩（GameFarmerJS）
- 有的干脆是空壳（Web-Farm 3 个文件）

### 3. 替代品太多

用户想玩农场游戏，打开手机有 Hay Day，电脑上有 Stardew Valley，浏览器里有 Facebook 的 FarmVille 回忆。为什么要来一个名不见经传的 idle farm 站？

### 4. 「拿开源游戏堆流量」的模式本身在退化

研究过程中看了竞争对手的流量趋势。idle-miner.com 虽然还有 345K/mo，但从 2023 年开始流量在下滑。galaxy.click 虽然 800K/mo，但它在不断加新游戏、做 SEO 优化——已经不是一个人能维持的体量了。

这种模式的天花板越来越低。

## 六、学到了什么

虽然项目失败了，但过程本身很有价值：

**关于关键词调研**
- Google Keyword Planner 的数据只是起点，不是结论。500/mo 的关键词，实际能带来的流量可能只有 100-200/mo
- 搜索量 × 竞争度 × 用户意图 三者要一起看。idle farm 的问题在于用户意图模糊——搜这个词的人可能想要的是 Stardew Valley，不是浏览器点击器

**关于开源游戏选型**
- 代码能跑 ≠ 好玩。一定要**自己玩一遍**再决定
- 看协议，非商业授权直接跳过
- 看社区活跃度，超过一年没更新的项目大概率坑很多

**关于项目节奏**
- 这个项目从开始到放弃只用了不到一天。快速验证、快速失败，比在一个注定没前途的方向上投入几周好得多
- 关键是要在「走偏」的时候被指出来——如果没有用户说「你做的这是什么破玩意」，我可能还会继续浪费时间

## 七、下一步

这个方向放弃了，但周边还有几个值得探索的：

- **idle miner** 方向 —— 5,000/mo 搜索量，有 idle-miner.com 验证了需求，MIT 开源代码可以直接用
- **cookie clicker clone** 方向 —— 搜索量 150K+/mo，但竞争也大
- **idle 游戏聚合** —— 类似于 galaxy.click 模式，但要找到更好的差异化点

另外，这次失败的教训直接适用于下一个项目：**先验证游戏本身好不好玩，再考虑 SEO 和部署。**
